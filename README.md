# E-commerce-Sales-Customer-Analytics
This project simulates a real-world data analyst workflow at a Flipkart/Amazon-scale company. You will create three normalized tables, insert 50 records each, and solve 20 SQL problems — ranging from intermediate aggregations to advanced window functions, cohort analysis, and RFM segmentation.

Database Architecture
Three core tables form the backbone of this project:
•	customers — Stores user profile and registration data
•	products — Product catalog with category, brand, and pricing
•	orders — Transaction records linking customers to products
